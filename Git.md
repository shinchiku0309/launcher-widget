# Git 管理ルール

このプロジェクトは次のリポジトリで管理します。

```text
http://192.168.1.102:3000/hiza-NAS/launcher.git
```

## 基本方針

- メインブランチは `main` とします。
- 作業前に `git status` を確認し、意図しない変更が混ざっていないことを確認します。
- 生成物、ローカル設定、Visual Studio の一時ファイルはコミットしません。
- コミットは小さく分け、1 コミット 1 目的を目安にします。
- README、設定、ビルド手順に影響する変更を行った場合は、関連ドキュメントも同じコミットで更新します。

## コミットメッセージ

コミットメッセージは内容が分かる日本語で記述します。必要に応じて Conventional Commits 風の接頭辞を使います。

例:

```text
docs: README と Git 管理ルールを追加
feat: ボタン編集ダイアログを追加
fix: URL 起動時の引数処理を修正
```

## push 前の確認

push 前に次を確認します。

- `git status --short` で不要な差分がないこと。
- ビルドに関わる変更をした場合は Visual Studio 2022 の `x64` / `Release` でビルド確認すること。
- ドキュメント変更のみの場合は、誤字、パス、コマンド例が実態と一致していること。

## リモート設定

初回のみ次の設定を行います。

```text
git remote add origin http://192.168.1.102:3000/hiza-NAS/launcher.git
git branch -M main
```

以降は `main` ブランチへ push します。
